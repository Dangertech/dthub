  • A Voxel game in which you build a society from scratch
    The Idea is very rough, but it is supposed to be a city builder/simulation which takes on progressively larger scale. You are an immortal being with otherwise human powers that lives since the 21st Century, now finally pulling itself up to rebuild the universe. Then you develop spacefaring, extending to other planets, eventually conquering entire galaxies. You complete the game by building an artifact that can create another universe, and create a new savefile.

I've gotten a Raspberry Pi 4 Model B recently and am currently collecting ideas on what to do with it. I want to use it as a server and have it running 24/7, so it will probably complete data collection and web scraping tasks. My ideas are pretty rough at the moment.

  • A simple log tracking the status of the pi and the devices I connect to it
    This is a must have. Also, I want to log my SSH accesses but I don't see the ssh log under /var/log/auth.log?
  • A Taskserver
    I am using Taskwarrior, a really cool and featureful CLI to-do list. They also have this project called Taskserver which lets you, as the name suggests, set up a tool on a server that synchronizes all your tasks across all your devices. I currently do this through OneDrive by just having my task folder in my OneDrive directory that I sync regularly with the amazing OneDrive CLI tool by abraunegg, but having a synchronization via my very own server sounds way cooler.
  • A camera system with face detection
    I've always wanted to play around with OpenCV. I'd simply connect a webcam to the pi and let a program run that collects the webcam data, saves an image every 3 seconds or so and tries to get interesting data out of it (how bright is it, are there faces/movement on the image, etc). I don't know if it's a smart idea to pull that off with C++ as well, maybe I'd have to use Python for that, but I like the idea to try and decode the camera data through C++ by myself.

I want the pi to be able to start all it's tasks autonomously, so I don't have to plug in a monitor and keyboard and start them manually every time it boots. This way, I have a little box that I just have to supply with power to start it. I still have to configure a static IP address for the wi-fi chip though.
