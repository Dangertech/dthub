So my family has gifted me a Raspberry Pi 4 Model B for Christmas and they are slowly realizing that this was a horrible idea and they are slowly stripped of their privacy and confidence that no cameras are installed in this household. (I'm just joking of course, right? Right!?)
I am working on software and scripts, published and unpublished, that are aiming to make it an information collection center that runs routine tasks, like backing up, making my data more accessible to me and gathering useful intel on stuff on the internet and my surroundings.

  • A simple log tracking the status of the pi and the devices I connect to it
    This is a must have. Important data should be saved to a file on the pi, like which temperature it has, when it got disconnected from the internet (my father fears this thing, so this might be very useful), or maybe even if it crashes, but I don't know if I can get runit to do something if it fails. Also, I want to log my SSH accesses but I don't see the ssh log under /var/log/auth.log?
    I have already written the backend for this, userlog
  • A Taskserver
    I am using Taskwarrior, a really cool and featureful CLI to-do list. They also have this project called Taskserver which lets you, as the name suggests, set up a tool on a server that synchronizes all your tasks across all your devices. I currently do this through OneDrive by just having my task folder in my OneDrive directory that I sync regularly with the amazing OneDrive CLI tool by abraunegg, but having a synchronization via my very own server sounds way cooler.
  • A camera system with face detection
    I've always wanted to play around with OpenCV. I'd simply connect a webcam to the pi and let a program run that collects the webcam data, saves an image every 3 seconds or so and tries to get interesting data out of it (how bright is it, are there faces/movement on the image, etc). I don't know if it's a smart idea to pull that off with C++ as well, maybe I'd have to use Python for that, but I like the idea to try and decode the camera data through C++ by myself.

My current setup starts working autonomously as soon as I give power to it. I've set up a static IP address for my Ethernet connection which can be accessed about 5 seconds after powering on, thanks to the fabulous runit init scheme. However, the WiFi doesn't want to work for some reason which makes me dependent on having the Pi next to our router, where my father can reach out and easily switch it off because he's creeped out by it. I need to find a solution and make the Pi Ethernet-independent but I get an uneasy feeling that the WiFi chip might be broken...
